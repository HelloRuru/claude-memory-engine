# /記憶健檢 — 記憶檔案健康狀態

檢查所有記憶檔案和 hooks 的健康狀態。

## 步驟

1. **掃描記憶目錄**：`~/.claude/projects/{project-id}/memory/`
2. **掃描 session 目錄**：`~/.claude/sessions/`
3. **掃描日記目錄**：`~/.claude/sessions/diary/`
4. **掃描 learned 目錄**：`~/.claude/skills/learned/`
5. **檢查 debug log**：`~/.claude/sessions/debug.log`

## 輸出格式

```
記憶健檢報告

MEMORY.md 水位：{行數}/200（{百分比}%）{安全/警告/危險}

記憶檔案：
| 檔案 | 行數 | 最後更新 | 狀態 |
|------|------|---------|------|
| {filename} | {lines} | {date} | {OK/過期/太大} |

Session 紀錄：共 {N} 筆，最近：{date}
日記紀錄：共 {N} 篇，最近：{date}
踩坑紀錄：共 {N} 筆，最近：{date}
Learned Skills：共 {N} 個

Hooks 狀態：
- session-start.js: {OK/異常}
- session-end.js: {OK/異常}（debug.log 最後訊息：{msg}）
- memory-sync.js: {OK}
- write-guard.js: {OK}
- pre-push-check.js: {OK}
```

## 判斷標準
- MEMORY.md < 170 行 → 安全
- MEMORY.md 170-200 行 → 警告，建議搬家
- MEMORY.md > 200 行 → 危險，超出的記憶會被截斷
- 記憶檔超過 30 天沒更新 → 過期
- 記憶檔超過 200 行 → 太大
