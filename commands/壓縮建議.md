# /壓縮建議 — 什麼時候該壓縮 Context

幫你判斷目前的對話狀態適不適合用 `/compact`。

## 該壓縮的時候

| 情境 | 壓縮？ | 原因 |
| :--- | :----- | :--- |
| 研究/探索做完了，準備動手 | 是 | 研究的 context 很大，計畫才是精華 |
| 規劃完成，要開始寫程式 | 是 | 計畫已經在 TodoWrite 或檔案裡了 |
| Debug 結束了 | 是 | 除錯紀錄會汙染下個任務的 context |
| 切換到完全不同的任務 | 是 | 清除不相關的 context |
| 部署完成，開始新功能 | 是 | 部署紀錄不需要保留 |

## 不該壓縮的時候

| 情境 | 原因 |
| :--- | :--- |
| 正在寫程式碼的中途 | 會丟失變數名、檔案路徑、部分狀態 |
| 正在 debug 但還沒解決 | 會丟失錯誤訊息和嘗試過的方案 |
| 多檔案重構進行中 | 會忘記哪些檔案還沒改 |
| 剛讀完重要文件準備引用 | 文件內容會被摘要掉 |

## 自動建議

當 context 使用率超過 60% 時，主動建議目前的任務狀態適不適合壓縮。
