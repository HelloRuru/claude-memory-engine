# /大健檢 — 完整稽核

徹底掃描整個記憶生態系統。建議每週或大改動後執行。

## 用法

- `/大健檢` — 全面稽核（預設）
- `/大健檢 指令` — 只深度稽核指令層
- `/大健檢 記憶` — 只深度稽核記憶檔案
- `/大健檢 git` — 只檢查所有專案的 git repo 狀態
- `/大健檢 {專案名}` — 針對特定專案做完整稽核

包含 `/健檢`（小健檢）的全部項目，加上以下額外檢查：

## 額外檢查

### 7. 指令層
- 列出所有指令，標記每個指令的功能
- 偵測重疊的指令（功能類似但名稱不同）
- 驗證指令中引用的檔案路徑是否都還存在
- 標記過時的指令（引用已刪除的檔案）

### 8. 記憶檔案稽核
- 每個記憶檔：有沒有被 MEMORY.md 索引？（找孤兒）
- 每個 MEMORY.md 條目：目標檔案存在嗎？（找斷鏈）
- 跨檔案重複內容偵測
- 所有檔案行數（標記超過 300 行的肥胖檔案）
- 驗證內部路徑引用是否還正確

### 9. 交叉引用完整性
- 完整鏈：MEMORY.md → memory/*.md → skills → references/
- 檢查 CLAUDE.md 和 MEMORY.md 之間有沒有重複的規則（應該只住一邊）
- 檢查指令和技能之間有沒有衝突的觸發詞

### 10. 環境設定
- `~/.claude/settings.json` — hooks 設定存在嗎？
- `~/.claude/scripts/hooks/` — 所有 hook 腳本存在且可讀？
- `~/.claude/sessions/` — session 目錄存在嗎？
- `~/.claude/skills/learned/` — 技能目錄存在嗎？

### 11. Git Repo 狀態
- 掃描已知的 repo（從專案設定讀取）
- 每個 repo：有沒有未 commit 的改動？落後 remote 嗎？
- 標記有 dirty working tree 的 repo

## 輸出格式

```
=== 大健檢報告 ===

--- 基礎（同小健檢）---
{小健檢的完整輸出}

--- 指令層 ---
- commands/: {N} 個指令
- [OK] diary.md — 反思日記
- [WARNING] {檔案} — 功能與 {另一個檔案} 重疊
- [BROKEN] {檔案} — 引用了不存在的路徑

--- 記憶檔案 ---
- memory/: {N} 個檔案，共 {total} 行
- [OK] {檔案}（{N} 行）— 已被 MEMORY.md 索引
- [ORPHAN] {檔案} — 不在 MEMORY.md 索引中
- [FAT] {檔案} 有 {N} 行，建議拆分

--- 交叉引用 ---
- [OK] MEMORY.md → memory/ → skills → references/ 鏈完整
- [DUPLICATE] 「{規則}」同時出現在 CLAUDE.md 和 MEMORY.md
- [CONFLICT] 觸發詞「{詞}」同時在 {指令 A} 和 {技能 B} 出現

--- 環境設定 ---
- settings.json: OK（hooks 已設定）
- hooks/: {N} 個腳本
- sessions/: OK
- skills/: OK

--- Git Repo ---
- [CLEAN] {repo} — 已與 remote 同步
- [DIRTY] {repo} — 有 {N} 個未 commit 的檔案
- [BEHIND] {repo} — 落後 remote {N} 個 commit

建議：
1. ...
2. ...
```

## 注意事項

- 列出所有發現和建議，但不自動修改任何檔案
- 等使用者確認再動手
- 報告要好掃描 — 統一用 [OK]、[WARNING]、[BROKEN] 標籤
