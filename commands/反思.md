# /反思 — 反思分析

分析最近的日記和 session 紀錄，找出規律，提出 CLAUDE.md 或 MEMORY.md 的更新建議。

## 步驟

1. **讀最近的日記**：掃描 `~/.claude/sessions/diary/` 最近 7 天的日記
2. **讀 session 摘要**：掃描 `~/.claude/sessions/` 最近的 session 紀錄
3. **讀踩坑紀錄**：掃描 `~/.claude/skills/learned/auto-pitfall-*.md`
4. **分析規律**：
   - 使用者最常做的工作類型是什麼？
   - 哪些踩坑重複出現？應該寫進 CLAUDE.md 避免嗎？
   - 有沒有新的偏好或規則要記下來？
   - 有哪些 MEMORY.md 的索引過時了？
5. **提出建議**：列出具體的更新建議，讓使用者確認後再執行

## 輸出格式

```
反思報告（{日期範圍}）

工作統計：
- 共 X 次 session，最常碰的專案：{名稱}
- 最常用的工具：{列表}

發現的規律：
1. {規律描述} → 建議：{動作}
2. ...

踩坑統計：
- 重複踩坑：{描述} → 建議寫進 CLAUDE.md
- 已解決不再出現：{描述} → 可以從 learned/ 歸檔

建議更新：
- [ ] CLAUDE.md 加入：{新規則}
- [ ] MEMORY.md 更新：{哪個索引}
- [ ] 移除過期紀錄：{哪些檔案}
```

## 注意事項
- 只提出建議，不自動修改 CLAUDE.md（要使用者確認）
- 白話文報告，一眼看懂
