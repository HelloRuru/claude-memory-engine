# /學習 — 踩坑自動學習

在工作過程中自動偵測並儲存踩坑經驗。

## 什麼時候該觸發（Claude 自己判斷）

1. **踩坑後解決** — 試了錯誤的方向，最後找到正確解法
2. **非直覺的 workaround** — 瀏覽器怪癖、函式庫限制、平台特殊行為
3. **路徑/環境陷阱** — Windows 路徑問題、`~` 解析錯誤、跨平台差異
4. **部署踩雷** — 部署平台的坑、快取問題、DNS 延遲
5. **花超過 3 次嘗試才解決** — 不容易解決的問題值得記

## 不需要學的

- 已經在 CLAUDE.md 或專案文件裡的設定
- 一般的 CSS/HTML 寫法（搜尋就有的）
- 只在這次有用、不會再遇到的一次性修正

## 存檔格式

存到 `~/.claude/skills/learned/{pattern-name}.md`：

```markdown
# {描述性標題}

**日期：** {今天}
**專案：** {哪個站}

## 問題
{什麼情境}

## 解法
{怎麼修的}

## 下次遇到時
{什麼情境該想到這個}
```

## 行為

- 存完後簡短告訴使用者：「學到了：{標題}」
- 不要長篇大論解釋為什麼要記
- 安靜地學，不打斷工作節奏
