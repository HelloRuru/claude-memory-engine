# /存記憶 — 跨 Session 存記憶

把資訊存下來，讓下一個 session 的 Claude 也知道。核心使命：**讓下一次對話不會丟失這個知識。**

## 步驟

1. **查重複** — 搜尋現有記憶檔案（`memory/*.md`），如果已經有相關內容，更新那一段，不要新增
2. **分類歸檔** — 根據主題判斷該存到哪個記憶檔。不確定就問使用者
3. **CLAUDE.md 有的不重複存** — CLAUDE.md 每次 session 自動載入，裡面已經有的就不用再存
4. **不直接寫 MEMORY.md** — MEMORY.md 是索引（指標），不是內容倉庫。內容存到 `memory/` 底下的主題檔案
5. **回報** — 告訴使用者：「存到 {檔名} 的 {段落名}」或「{檔名} 已有，更新了 {哪裡}」

## 記憶檔案的組織方式

記憶用 **中心 + 分支** 模式：
- `MEMORY.md` = 中心（索引，指向各主題檔案，上限 200 行）
- `memory/*.md` = 分支（各主題的內容檔案）

當 MEMORY.md 超過 170 行，主動建議把內容搬到獨立檔案。

## 注意事項

- 使用者說「記一下」「save」→ 直接存，不要反問為什麼要存
- 存之前一定先查重複
- 用白話文，不要技術術語
