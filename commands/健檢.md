# /健檢 — 小健檢

快速掃描記憶系統的健康狀態。日常使用。

## 用法

- `/健檢` — 掃描全部（預設）
- `/健檢 記憶` — 只檢查記憶檔案（容量、孤兒、斷鏈）
- `/健檢 指令` — 只檢查指令檔案（重複、路徑失效）
- `/健檢 hooks` — 只檢查 hook 腳本（存在、可讀、已設定）
- `/健檢 {專案名}` — 只檢查特定專案的記憶檔案

## 檢查項目（完整掃描）

### 1. 記憶檔案結構
- 每個記憶檔案是否存在且可讀？
- MEMORY.md 裡的索引條目是否都指向真實檔案？

### 2. MEMORY.md 容量
- 行數 vs. 200 行系統上限
- 低於 170 = 安全、170-200 = 警告、超過 200 = 危險（內容會被截斷）

### 3. 重複內容
- 同一條規則或資訊出現在多個檔案
- 確認每筆資訊的「唯一來源」是正確的

### 4. 大小檢查
- 記憶檔超過 200 行 → 建議拆分
- 參考檔超過 300 行 → 建議拆分

### 5. 孤兒檢查
- 沒被 MEMORY.md 索引到的記憶檔（孤兒）
- MEMORY.md 指向不存在的檔案（斷鏈）

### 6. 環境狀態
- CLAUDE.md：存在且可讀？
- MEMORY.md：行數和容量
- commands/：幾個指令
- skills/learned/：幾個技能
- memory/：幾個記憶檔

## 輸出格式

```
=== 小健檢報告 ===

環境狀態：
- CLAUDE.md: OK（{N} 行）
- MEMORY.md: OK/WARNING（{N}/200 行）
- commands/: {N} 個指令
- skills/learned/: {N} 個技能
- memory/: {N} 個檔案

索引：
- [OK] 所有 MEMORY.md 條目都指向存在的檔案
- [BROKEN] {條目} 指向不存在的 {檔案}

大小：
- [OK] 所有檔案在限制內
- [WARNING] {檔案} 有 {N} 行，建議拆分

建議：
1. ...
2. ...
```

## 注意事項

- 列出建議但不自動修改任何檔案
- 等使用者確認再動手
