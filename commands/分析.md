# /分析 — 記錯題

比對使用者修改了你的哪些輸出，找出哪裡錯了、為什麼錯。

## 步驟

1. **找差異** — 找出使用者改了你的哪些內容（文字、程式碼、結構、語氣）
2. **對照規則** — 每個修改對照 CLAUDE.md、記憶檔案、learned skills 裡的既有規則
3. **記漏掉的規則** — 規則存在但你沒遵守，記下來並標次數（例如「第 2 次漏掉」）
4. **歸納新模式** — 修改對不上任何既有規則，歸納成新原則
5. **存進錯題本** — 寫到 `~/.claude/skills/learned/writing-review-list.md`（或專案專屬的檔案）
6. **回報** — 告訴使用者：「記了 N 條訂正。M 條是已有規則沒做到，K 條是新模式。」

## 錯題本格式

```markdown
### 規則名稱
- **規則：** 該怎麼做
- **漏掉：** N 次
- **狀態：** active / cleared
- **例子：** 錯的 → 對的
```

## 什麼時候打

改完馬上打最好。context 越新鮮，分析越準。
