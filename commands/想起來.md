# /想起來 — 災難恢復

本地記憶檔案遺失或損毀時，從 GitHub 備份還原。

## 前提

你需要先有一個 GitHub repo 作為記憶備份（用 `/備份` 或 `/同步` 推上去的那個）。如果還沒建立備份 repo，這個指令無法使用。

## 什麼時候用

- 換到新電腦
- 本地記憶檔案被意外刪除
- SessionStart hook 壞了
- 記憶檔案看起來是空的或損毀

## 步驟

1. **保護目前狀態** — 先執行 `/同步`，推送任何剩餘的本地變更
2. **從 GitHub 拉回** — Clone 或 pull 你的 GitHub 記憶 repo（`~/.claude/claude-memory/` 或你設定的路徑）
3. **驗證** — 讀 MEMORY.md，掃描所有索引條目
4. **回報** — 「想起來了！找到 {N} 個索引、{M} 個記憶檔案」

## 注意事項

- 正常情況下，SessionStart hook 會自動載入上次摘要 + 最近的記憶變更。不需要常用這個指令
- 這是「打破玻璃」的緊急選項，自動化系統壞了才用
- 拉之前一定先推，避免覆蓋本地變更
