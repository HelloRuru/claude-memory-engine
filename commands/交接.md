# 交接（Session Handoff）

產生交接檔案，讓另一個 Claude Code session 知道這邊在做什麼、做到哪裡、還有什麼沒做。

## 步驟

1. **分析目前工作** — 這個 session 在做什麼？改了哪些檔案？做了什麼決策？

2. **寫交接檔案** — 存到專案的記憶目錄，檔名 `handoff-{主題}.md`，格式如下：

```markdown
---
name: handoff-{主題}
description: {一行摘要}
type: project
---

# {標題} 交接

**狀態：** {做完了什麼 / 還有什麼待做}
**日期：** {今天}

## 做了什麼
{已完成工作的重點}

## 還沒做的
- [ ] {待做 1}
- [ ] {待做 2}

## 關鍵決策
{下一個 session 需要知道的決策}

## 改了哪些檔案
{這個 session 改過的檔案清單}
```

3. **回報** — 告訴嚕寶交接檔案已存好，下一個 session 開始時會自動偵測。

## 運作方式

- **SessionStart hook** 自動讀取所有未讀的 `handoff-*.md`，新對話一開始就注入
- **memory-sync hook** 對話中偵測到新的交接檔案出現，立刻提示
- 用 `.handoff-read.json` 追蹤已讀狀態，每份交接只顯示一次
